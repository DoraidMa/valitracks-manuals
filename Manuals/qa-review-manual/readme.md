<h2>README</h2>
<p>This guide explains how to start QA project management inside Valitracks. It covers:</p>
<ul>
    <li>Accessing the project from the <strong>Managed Projects</strong> tab</li>
    <li>Entering the workflow through the <strong>Project Offering (ERP) module</strong> using the four-squares icon</li>
    <li>Using the <strong>Project Tracker</strong> to verify access, task allocation, progress, and tasking communication</li>
    <li>Checking invoices against purchase orders and ensuring invoicing is enabled only after project completion</li>
    <li>Requesting AI translation access when managing unfamiliar languages</li>
    <li>Using the workspace-specific <strong>noting system</strong> to document QA observations</li>
    <li>Reviewing and overseeing the <strong>LV system</strong> (harmonization, BT review, infill review, eCOA)</li>
</ul>
<p>This manual is intended for QA reviewers responsible for full oversight of the project workflow across all relevant Valitracks modules.</p>
