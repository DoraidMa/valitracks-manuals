# How to Start the Recruiting Process Using the ERP Workflow

This interactive guide explains how to start recruiting patients immediately after the study is launched using the ERP (Project Offer & Workflow) system in Valitracks.

The simulation walks you through:

### **1. Accessing the Study**
- Navigate to **My Pages â†’ My Offers**.
- Locate the study with the recruiting component enabled.
- Accept the project offer.

### **2. Connecting Recruitment to the Workflow**
- Understand where recruiting is positioned inside the ERP workflow.
- See how task visibility is triggered after accepting the offer.
- Learn the difference between *recruiter tasks* and *in-vivo interviewer tasks*.

### **3. Starting the Recruiting Module**
- Go to **My Interviews** to view the assigned recruiting project.
- Click the **4-square icon** to access the recruiting simulation.
- Begin the step-by-step process of screening and onboarding patients before the target-language questionnaire is finalized.

### **4. What Recruiters Should Prepare**
- How to proceed when harmonization is not yet completed.
- Understanding what data recruiters collect.
- Preparing the patient list for interviewers once the COA is ready.

### **5. What Happens Automatically in Valitracks**
- Automatic linking of recruiting tasks to the study.
- Status propagation to PMs and team leaders.
- Visibility of progress inside PM Dashboard and Workflow page.

---

## **Purpose of This Guide**
Recruiting often begins **before** the final target-language questionnaire is ready.  
This manual ensures that recruiters and interviewers understand what they must complete during this early phase, and how the ERP workflow triggers these tasks correctly upon study launch.
